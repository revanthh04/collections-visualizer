🌳 The Array-Backed Tree

A PriorityQueue looks like a tree, but it is actually stored in a flat array! Math is used to find relatives:

  • Left Child: Always at index 2i + 1.
  • Right Child: Always at index 2i + 2.
  • Parent: Always at index (i - 1) / 2.

⚖️ The Heap Property

It is not a sorted list. Instead, it maintains the "Heap Property":

  • Min-Heap: Every parent is smaller than its children. The smallest element is always exactly at the root (index 0).
  • Max-Heap: The opposite. Every parent is larger than its children, meaning the largest element is at the root.

🔄 Heapification (Up & Down)

To maintain the structure when adding or removing elements, the heap restructures itself:

  • Sift Up (add): New elements are added to the very end of the array, then bubble up if they are smaller than their parent.
  • Sift Down (poll): We remove the root, move the last element to the root, and bubble it down, swapping with its smallest child.

⚙️ Common Methods & Time Complexity

Method Description Process Time Comp.
boolean add(E e) Inserts the element into the priority queue. Insert at end, Sift Up. O(log N)
E poll() Retrieves and removes the head of this queue. Remove root, move tail to root, Sift Down. O(log N)
E peek() Retrieves, but does not remove, the head of this queue. Return array[0]. O(1)
boolean remove(Object o) Removes a single instance of the specified element. Find O(N), remove, Sift Up/Down. O(N)

* Note: Java's PriorityQueue is NOT thread-safe. Use PriorityBlockingQueue for concurrent access.
