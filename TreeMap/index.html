🌲 Red-Black Tree

TreeMap stores key-value pairs in a self-balancing Red-Black Tree:

  • Keys are sorted by their natural ordering (Comparable) or a custom Comparator
  • All operations are O(log n) — guaranteed by the balanced tree height
  • In-order traversal gives keys in ascending sorted order
  • Unlike HashMap, TreeMap does NOT use hashing — it uses key comparisons

🧭 NavigableMap Methods

TreeMap implements NavigableMap, providing powerful navigation:

  • firstKey() / lastKey() — smallest / largest key
  • floorKey(k) — greatest key ≤ k
  • ceilingKey(k) — smallest key ≥ k
  • lowerKey(k) / higherKey(k) — strictly less/greater
  • subMap(), headMap(), tailMap() — range views

☕ Java Caveats

  • Keys must be Comparable or a Comparator must be provided at construction
  • TreeSet is backed by TreeMap (elements are keys with dummy values)
  • Not thread-safe: use Collections.synchronizedSortedMap() or ConcurrentSkipListMap
  • Use when you need sorted iteration or range queries (subMap, headMap, tailMap)
  • Slower than HashMap for simple get/put — only use when ordering matters
