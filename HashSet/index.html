#️⃣ Hash Table Array

A HashSet stores elements in an array of "buckets". The index is computed using a hash function:

  • Hash code maps the object to an integer
  • Modulo/Masking hash & (capacity - 1) maps the hash to an array index
  • Provides O(1) average time complexity for access, insert, and delete!

⚡ Collision & Separate Chaining

When two elements hash to the same bucket:

  • Separate Chaining: Multiple elements are stored in a linked list within the same bucket
  • Each node has a next pointer → forms a chain
  • Java 8+ Treeification: When a chain exceeds 8 nodes, it converts to a Red-Black Tree
  • When elements are removed and chain drops ≤ 6, it untreeifies back to a linked list

📈 Load Factor & Rehashing

As the set fills up, collisions become too frequent. Load Factor = size / capacity.

  • Default Load Factor Threshold: 0.75 (75% full)
  • Rehashing: When threshold is exceeded, Java creates a new array 2x the size
  • All elements are re-hashed and moved, keeping chains short and restoring O(1) performance

☕ Java HashSet Caveats

  • Backed internally by a HashMap (elements are keys, values are dummy PRESENT objects)
  • Unordered: Iteration order is unpredictable and changes after rehash
  • Accepts exactly one null element
  • Not thread-safe — use ConcurrentHashMap.newKeySet()
  • Must override hashCode() and equals() properly
