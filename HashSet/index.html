How Hashing Works

The element's hashCode() is processed to determine a bucket index: index = hash(e) & (capacity - 1). This uses bitwise AND instead of modulo for efficiency (capacity is always a power of 2).

Collision Handling (Open Addressing)

When two elements hash to the same bucket, we use Linear Probing. If bucket [i] is full, we look at [i+1], then [i+2], wrapping around until an empty slot is found.

Load Factor & Rehashing

Default load factor is 0.75. When size > capacity × 0.75, the table rehashes — capacity doubles and all elements are redistributed. This prevents probing sequences from becoming O(n).
