🌲 Binary Search Tree (BST)

A Binary Search Tree is a node-based data structure where each node has at most two children. It enforces a strict ordering:

  • Left subtree — all values are < parent
  • Right subtree — all values are > parent
  • In-order traversal always produces sorted output
  • Search, insert, delete are O(h) where h = height

⚠️ Problem: A plain BST can become degenerate (essentially a linked list) with sorted input, making all ops O(n) worst case!

🔴 Red-Black Tree — Self-Balancing BST

A Red-Black Tree is a BST with an extra color bit per node (Red or Black) and specific rules that keep the tree approximately balanced, guaranteeing O(log n) for all operations.

Java's TreeSet and TreeMap are backed by a Red-Black Tree internally. It was invented by Rudolf Bayer (1972) and refined by Leonidas Guibas & Robert Sedgewick (1978).

Unlike AVL trees, RBTs allow slightly less perfect balance in exchange for fewer rotations during insertions & deletions, making them ideal for write-heavy workloads.

📜 The 5 Red-Black Tree Rules (Invariants)

1 Every node is either RED or BLACK.
2 The root is always BLACK. After every insert fix-up, we ensure root.color = BLACK.
3 Every leaf (null / NIL) is considered BLACK. These are implicit sentinel nodes.
4 No two consecutive RED nodes — if a node is RED, both its children must be BLACK. (No red-red parent-child pair)
5 Every path from a node to any of its descendant null leaves contains the same number of BLACK nodes (called "black-height"). This is the key balancing property!

🔄 Rotations — The Balancing Tool

When an insertion or deletion violates the RBT rules, the tree self-corrects using rotations and recoloring.

Left Rotation
x y \ → / y x

x goes down-left, y goes up

Right Rotation
y x / → \ x y

y goes down-right, x goes up

At most 2 rotations for insert, 3 rotations for delete. Recoloring may propagate up the tree.

☕ Java TreeSet Internals

In Java, TreeSet is a wrapper around TreeMap. Each element is stored as a key in the internal TreeMap with a dummy value (PRESENT).

  • Implements NavigableSet → SortedSet → Set
  • Elements must be Comparable or a Comparator must be provided
  • Rejects duplicates — add() returns false if element already exists
  • Does NOT allow null (throws NullPointerException)
  • Not thread-safe — use Collections.synchronizedSortedSet() or ConcurrentSkipListSet
  • Unlike HashSet (O(1) ops, unordered), TreeSet guarantees sorted order at the cost of O(log n) per operation
