🔗 Doubly-Linked List

LinkedHashMap extends HashMap and adds a doubly-linked list across all entries:

  • Each Entry has before and after pointers
  • A head points to the oldest entry, tail to the newest
  • Insertion order is preserved — iterating gives you entries in the order they were first inserted
  • Slight memory overhead per entry (~2 extra pointers)

🔄 Access-Order Mode (LRU Cache)

LinkedHashMap supports an access-order mode — the secret behind LRU caches:

  • new LinkedHashMap(cap, lf, true) — 3rd param enables access-order
  • Every get() or put() call moves the entry to the tail (most recently used)
  • Override removeEldestEntry() to auto-evict the oldest entry when size exceeds a limit
  • This is how LRU caches are built in Java — a classic interview question!

☕ Java Caveats

  • Extends HashMap: Same initial capacity (16), load factor (0.75), rehashing logic
  • Slightly slower than HashMap due to linked list maintenance
  • Not thread-safe: use Collections.synchronizedMap()
  • Iteration is O(size), NOT O(capacity) — faster than HashMap iteration!
  • Perfect for maintaining a predictable iteration order while keeping O(1) access
