🔗 HashSet + Doubly-Linked List

A LinkedHashSet provides all the benefits of a HashSet, but with predictability:

  • Internally uses a Hash Table for O(1) average lookups and mutations.
  • Crucial Addition: Every entry is also a node in a Doubly-Linked List.
  • This linked list naturally acts as a timeline, recording the exact sequence in which items were inserted.

⏱️ Insertion Order vs Access Order

  • Insertion Order (Default): Iterating returns elements in the exact order they were first added. Re-inserting an existing element does NOT change its position.
  • Backed internally by a LinkedHashMap.
  • Perfect for cases where you want to remove duplicates from a list but must not scramble the order!

⚡ Performance & Memory Overhead

The predictability comes at a small cost:

  • Memory: Higher overhead. Every node has two extra pointers (before and after).
  • Iteration: O(Size). This is actually faster than HashSet's O(Capacity + Size) because it walks the linked list directly instead of scanning empty buckets!

🔄 How Rehashing Works

When the load factor (0.75) is reached, the capacity doubles.

  • The array of buckets is rebuilt.
  • Rehashing walks through the Linked List to redistribute elements into the new buckets.
  • The linked list structure is preserved perfectly, so iteration order remains identical before and after a rehash!

⚙️ Common Methods & Time Complexity

Method Description Origin Time Comp.
boolean add(E e) Adds the specified element if it is not already present, appending it to the insertion list. Collection O(1)*
boolean remove(Object o) Removes the element from the hash table and unlinks it from the list. Collection O(1)*
boolean contains(Object o) Returns true if this set contains the specified element. Collection O(1)*
Iterator<E> iterator() Returns an iterator over the elements in insertion order. Iterable O(1) (to get)

* Average case. LinkedHashSet operations have slightly higher constant factor over HashSet due to maintaining the doubly-linked list. Iterating is O(Size), not O(Capacity).
