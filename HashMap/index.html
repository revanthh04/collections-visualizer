#️⃣ Hash Table Array

HashMap stores entries (key-value pairs) in an array of "buckets" called Node<K,V>[] table:

  • Hash code of the KEY determines the bucket index
  • Bucket Index: hash(key) & (capacity - 1) — a bitwise AND for speed
  • Default initial capacity is 16, always a power of 2
  • Each entry stores: hash, key, value, next

⚡ Collision Handling

When two keys hash to the same bucket, HashMap uses separate chaining:

  • Linked List: Multiple entries chain via next pointers
  • Java 8+ Treeification: When a chain exceeds 8 nodes, it converts to a Red-Black Tree (O(log n) lookup)
  • When the tree shrinks below 6 nodes, it converts back to a linked list
  • Good hash distribution keeps chains short → O(1) amortized

📈 Load Factor & Rehashing

Load Factor = size / capacity. When threshold is exceeded → rehash!

  • Default Load Factor: 0.75 (75% full)
  • Threshold: capacity × loadFactor (e.g., 16 × 0.75 = 12)
  • Rehashing: Create new array 2× the size, re-hash ALL entries
  • Rehashing is O(n) but happens rarely — amortized O(1) per put

☕ Java HashMap vs HashSet

  • HashSet is backed by HashMap internally — elements are keys with a dummy PRESENT value
  • HashMap allows one null key (stored at bucket 0) and multiple null values
  • Not ordered: iteration order is unpredictable
  • Not thread-safe: use ConcurrentHashMap for concurrency
  • Keys must override hashCode() and equals() properly
