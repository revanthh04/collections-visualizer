📦 Dynamic Array Backend

An ArrayList stores elements inside a contiguous array. When it runs out of space, it automatically grows:

  • Contiguous Memory ensures fast sequential iteration and CPU cache locality.
  • Initial Capacity: Starts at 10 (by default).
  • Growth Factor: Grows by 1.5x (newCap = old + (old >> 1)).

⚡ The O(1) Advantage

ArrayList is heavily optimized for reads and append-only writes:

  • Random Access: get(i) and set(i, val) are instantly calculated via memory offsets, making them O(1).
  • Appending: add(val) at the end is amortized O(1) (fast except during a resize).

📉 The O(N) Caveat

Because the array is contiguous, inserting or removing elements in the middle forces shifting:

  • Shifting Cost: add(i, val) and remove(i) take O(N) time because System.arraycopy must move elements.
  • Removing from the head (index 0) is always the worst case, O(N).

⚙️ Common Methods & Time Complexity

Method Description Origin Time Comp.
boolean add(E e) Appends the element to the end of the list. List O(1)*
void add(int i, E e) Inserts element at position i, shifting subsequent elements right. List O(n)
E remove(int i) Removes element at position i, shifting subsequent elements left. List O(n)
E get(int i) Returns the element at the specified position. List O(1)
E set(int i, E e) Replaces the element at the specified position. List O(1)

* Amortized complexity. It takes O(n) during the exact moment a resize happens, but effectively O(1) averaged over all insertions.
