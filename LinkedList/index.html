Internal Structure

Each node has three fields: prev, item, and next. The list maintains pointers to the first and last nodes, enabling O(1) operations at both ends.

Key Characteristics

  • addFirst / addLast: O(1) — direct pointer manipulation
  • get(index): O(n) — must traverse from nearest end
  • Implements: List, Deque, Queue, and can act as Stack
  • Memory: Each node stores 3 references (prev, item, next)
